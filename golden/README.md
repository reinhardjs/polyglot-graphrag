# Golden question sets (user-provided — CONFIDENTIAL, git-ignored)

Drop your quality-evaluation question sets in **this directory**. They are
**git-ignored** (`*.json` here is in `.gitignore`) so confidential
corpus content never gets committed to the repo.

## Why this dir

`evaluate_pipeline.py` and the release-gate's **Check 12 (Answer
quality)** score synthesized-answer quality (Faithfulness / Answer
Relevance / Context Precision / Context Recall) over a golden dataset.
That dataset is *your* corpus data — it must not be version-controlled.

## Format

A JSON list of objects:

```json
[
  {
    "question": "who reported BUG-204?",
    "ground_truth": "BUG-204 was reported by Alice.",
    "doc_id": "<source>::guidelines::incident-handling::incident-response-guide"
  },
  ...
]
```

Fields:
- `question` (required) — the query sent to `/ask`.
- `ground_truth` (optional but recommended) — a factual span used
  for Context Recall + Answer Relevance. If omitted, the harness
  falls back to the question itself for relevance scoring.
- `doc_id` (optional) — provenance; echoed in output, never required.
- `answer` / `contexts` (optional) — if present they're scored
  as-is; if absent, the gate/`-live` flag generates them by
  calling the running daemon's `/ask` (so they use whatever
  synthesis backend is live, e.g. E2B).

## How to run

```bash
# Score a set live (answers generated by the running daemon):
python evaluate_pipeline.py golden/<your-set>.json --live --domain enterprise

# The release gate uses golden.json by default:
python scripts/release-gate.py        # Check 12 reads golden/golden.json
```

## Override paths (env)

- `GOLDEN_DIR`  — directory holding golden sets (default: `golden/`).
- `GOLDEN_FILE` — filename inside `GOLDEN_DIR` (default:
  `golden.json`).

Example: `GOLDEN_FILE=my-corpus.json python scripts/release-gate.py`

The shipped demo golden `self-docs-demo.json` is built from the
auto-seeded self-docs corpus (`build_oragolden.py --out golden/self-docs-demo.json`)
and is safe to use for a smoke-test release gate without any confidential data:
`GOLDEN_FILE=self-docs-demo.json python scripts/release-gate.py`.

## IMPORTANT

**Do NOT commit `*.json` from this directory.** They contain
confidential corpus content. If you need to share a set, export it
out-of-band (not via git).
