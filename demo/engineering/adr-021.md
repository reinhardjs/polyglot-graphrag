# ADR-021: Adopt PostgreSQL as the primary transactional store

- Status: Accepted
- Date: 2026-03-14
- Deciders: alice (staff engineer), carol (db lead)

## Context

The checkout service previously used an in-house key-value store that could not
express relational integrity. Order, payment, and invoice rows drifted out of
sync during partial failures.

## Decision

Adopt **PostgreSQL 16** as the primary transactional store for the checkout
service. All writes go through a single primary; read replicas handle reporting.
The checkout service owns the `orders` and `payments` schemas.

## Consequences

- Strong consistency for order/payment state.
- Adds operational burden: backups, vacuum, connection pooling via PgBouncer.
- The billing service now reads order events via CDC from PostgreSQL.
