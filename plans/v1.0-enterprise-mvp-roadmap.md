## 1. Architecture Principles (what makes this "enterprise-grade")

These are non-negotiable for v1.0. Every phase must advance toward these:

1. **Domain-agnostic core** — add a domain by writing `domain_config.yaml` + a folder with `retrieve.py` and `ingest.py`. Nothing in `serve_gpu.py` or `federated.py` is domain-specific.

2. **Graceful degradation** — missing domain, dead Neo4j, empty Qdrant collection, or failed E4B synthesis must never 500. Return partial results with `"degraded": true` and a `"notice"` explaining what was unavailable.

3. **One request, one response** — `/ask` is the only endpoint a consumer needs. Domain, mode, synthesis, confidence floor, and dual-evidence preference are parameters on the same call.

4. **Provenance in every result** — every context in `/ask` carries `signal`, `concept_id`, `confidence`, `doc_id`, `chunk_idx`. No context without a source.

5. **Try-first documentation** — every domain ships with a QUICKSTART that includes copy-pasteable curl commands. A user new to the system runs one command and gets a result.

6. **Horizontal scale for retrieval** — retrieval (embed + search) must be parallelizable across domains. The federation layer runs retrievers concurrently; adding domains adds parallelism, not latency.

## 3. Delivery Phases (7 days)

### Phase 1 — Core Performance (Days 1-2)

**Objective**: Fix retrieval pipeline bottlenecks so adding more domains doesn't compound slowness.

This phase implements 8 performance optimizations. Summary:

| # | Fix | Waste eliminated | Effort |
|---|-----|-----------------|--------|
| 1 | Pass precomputed `vec` to clinical_prose retriever → skip HTTP re-embed | 60-160ms | Medium |
| 2 | Merge `_idf_for_keyword` + `_concepts_for_keyword` into single Cypher | 30-60ms | Low |
| 3 | Remove redundant `_edge_boost` (subset of `_symptom_to_disorder`) | 20-40ms | Low |
| 4 | Module-level cached Neo4j driver + Qdrant client | 10-20ms | Low |
| 5 | Enable semantic cache for `synthesize=false` | 250ms (repeat queries) | Low |
| 6 | Improve `_names_batch` to include non-English fallback in one batch | 5-20ms | Low |
| 7 | Precompute SNOMED IDF lookup table (startup build, O(1) query-time) | 80-120ms per keyword query | Medium |
| 8 | Skip BGE reranker when `synthesize=false` and pool ≤ 10 (raw retrieval scores suffice) | 20-40ms | Low |

**Acceptance criteria**:
- `/ask (synthesize=false)` for "fever and rash" (single domain, cold) is < 250ms p95
- Multi-domain `domain=all` cold latency < 400ms p95
- Same contexts returned as before (ranking preserved where semantically equivalent; raw-score ordering acceptable for synthesize=false)
- Repeat queries hit cache regardless of `synthesize` flag (< 10ms)
- No new errors in `logs/daemon_gpu.log`

**Deliverables**:
- Patched `clinical_prose/retrieve.py` (vec-accepting retrieve signature)
- Patched `domains/__init__.py` (forward vec in get_retriever lambda)
- Patched `snomed/retrieve.py` (merged Cypher, removed _edge_boost, cached driver, improved _names_batch, precomputed IDF lookup)
- Patched `serve_gpu.py` (cache gate fix, conditional reranker skip)
- Timing benchmark script: `scripts/bench_ask.py` (baseline vs optimized)

### Phase 3 — Enterprise Hardening (Days 4-6)

**Objective**: Make the system safe to expose publicly. No 500s, no silent failures,
no unrecoverable states. Observable, debuggable, and documented.

#### Day 4-5: Error Boundaries + Graceful Degradation

| Hardening | Where | What |
|-----------|-------|------|
| Neo4j unavailable | `snomed/retrieve.py`, all graph retrievers | Return `[]` + log warning; `/ask` continues with prose-only results |
| Qdrant unavailable | `clinical_prose/retrieve.py`, all dense retrievers | Return `[]` + log warning; `/ask` continues with graph-only results |
| Domain not found | `domain_loader.py` | Return `{"error": "unknown_domain", "available": [...]}` with 200 (not 500) |
| Empty collection | All retrievers | Return 0 contexts gracefully; no crash |
| E4B synthesis unavailable | `serve_gpu.py` `/ask` | Return partial result with `"answer": ""`, `"degraded": true`, `"notice": "synthesis unavailable"` |
| GLiNER extraction failure | `extractor_gliner.py`, sliding_window | Catch KeyError, continue with remaining windows; never abort full ingest |
| Concurrent write conflict | `ingest.py`, Neo4j MERGE | Use MERGE (idempotent), catch constraint violations, log and continue |

**Deliverables**:
- Global error middleware in `serve_gpu.py` (catch unhandled exceptions → 200 with `error` + `degraded` flag)
- Domain-level try/except in `federated.py::assemble` (one failed retriever doesn't kill the others)
- `/health` endpoint extended: reports per-domain status, Neo4j connectivity, Qdrant connectivity
- Request ID header propagation (`X-Request-ID`) for log correlation

## 5b. Known Limitations (v1.0.0 — verified 2026-07-15)

These are documented, hardware/scoping-bound gaps. They are NOT code defects.

- **`default` alias.** The plan body says `default → healthcare` (P2.10) while
  §0/§2 say `default → snomed`. v1.0 implements `default: alias snomed` AND
  `healthcare: alias snomed` (identical retrieval). This is a doc inconsistency,
  not a functional gap.
- **`/domains` response shape.** P2.10 VERIFY expects a JSON list; the live
  endpoint returns `{"domains":[...], "default_domain":..., "count":...}`. The
  endpoint is correct; the VERIFY command in the plan was written for the wrong
  shape.
- **Neo4j/Qdrant-down graceful degradation** (P3.12) is verified by code-path
  simulation (per-domain try/except returns degraded + other domains), not by
  live `systemctl stop` of the shared infra on the reference host.

> **RESOLVED:** `synthesis=true` latency was p95≈10.4s (only healthcare's
> verbose differential). Fixed by tightening `prompts/clinical_dx.md` to a
> concise ≤5-candidate ranked differential (≤250 words). Post-fix 20x synthesis
> p95: healthcare 2.38s, enterprise 1.91s, legal 1.40s, fraud 1.18s — all
> under the 3s target, answers still a valid ranked differential.
