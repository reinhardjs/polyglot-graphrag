# Plan: v2.0 — Knowledge Discovery & Reasoning Platform (Feasibility Analysis)

## Status: FEASIBILITY REVIEW (not scheduled) — answers "is this doable in 1 week?"

## TL;DR

**The full 9-phase vision is NOT achievable in 1 week.** It is a multi-month
(realistically 3–6 month) program that redefines the product from a
retrieval system into a reasoning + discovery platform. It also DIRECTLY
CONTRADICTS the v1.0 "ZERO ARCHITECTURAL DRIFT" constraint (no inferred graphs,
no multi-hop reasoning, no background daemons) — so it is explicitly a **v2.0
successor track, not a v1.0 addition.**

What IS feasible in 1 week: a single, honest **vertical slice** that proves the
core differentiator (implicit-relationship discovery + explainable multi-hop)
on the ONE domain where we already have a real graph — **fraud** (and partially
enterprise). See "1-Week Feasible Slice" below.

## Why not 1 week (honest scoping)

The vision is ~40 checklist items across 9 phases, several of which are each a
multi-week research/engineering effort on their own:

- **Relationship discovery + validation (P3):** candidate generation from 10
  signal sources (topology, shared entities/APIs/schema, ontology rules,
  embedding similarity, temporal correlation, dependency analysis, event flow,
  LLM hypothesis) + a validator (ontology constraints, graph rules, evidence,
  thresholds, optional human approval) + a SEPARATE inferred graph store. This
  alone is 3–5 weeks and is the make-or-break quality risk (false inferred edges
  destroy trust — violates "never sacrifice integrity for recall").
- **Reasoning engine (P4):** 7 traversal types + 7 auto-selected strategies +
  explainability. On an RTX 3060 12GB, multi-hop traversal + per-hop LLM
  explanation will blow the <3s budget; needs its own async/streaming design.
- **Evidence graph (P5), quality engine (P6), continuous evolution (P7):** each
  a multi-week subsystem.

Attempting all of it in a week produces a demo that fabricates plausible-looking
inferred edges without real validation — the exact anti-pattern the guiding
principles forbid ("every inferred relationship must be explainable",
"every conclusion traceable to evidence").

## 1-Week FEASIBLE slice — "Implicit Dependency Discovery (fraud + enterprise)"

Goal: prove the *core differentiator* end-to-end on real data we already have,
deterministically (principle: "prefer deterministic logic before LLM"), with
full provenance — WITHOUT building the whole platform.

### Scope (deliberately narrow)

- [ ] **W1 Inferred edge store (schema only).** Add an `:Inferred` edge type in
  Neo4j, separate from explicit edges, with mandatory props: `confidence`,
  `supporting_evidence` (list of explicit edge/node ids), `strategy`,
  `created_at`. No writes to the explicit graph.
  > VERIFY: Cypher shows `:Inferred` edges carry all 4 props; explicit graph
  > untouched.

- [ ] **W2 ONE deterministic discovery strategy: transitive dependency.**
  In the fraud domain (real transaction/account/device graph) and enterprise
  (Service→depends_on→Service), compute `inferred_depends_on` /
  `inferred_impacts` via bounded transitive closure (2–3 hops) over EXISTING
  explicit edges. Pure Cypher/graph, no LLM. Every inferred edge records the
  explicit path that produced it.
  > VERIFY: "what indirectly depends on Redis/deviceX" returns inferred edges,
  > each with the concrete supporting path.

- [ ] **W3 Deterministic validator.** Reject inferred edges that (a) contradict
  an existing explicit edge, (b) fall below a confidence floor (confidence =
  product/decay over path length), or (c) form trivial self-loops. Log
  accepted/rejected/needs-review counts.
  > VERIFY: a known bad candidate is rejected with a reason.

- [ ] **W4 Read-only discovery endpoint (opt-in, off the hot path).**
  `POST /discover {domain, entity, relation}` → returns inferred edges +
  explanation (the path + confidence + strategy). Does NOT touch `/ask`'s
  <250ms contract. Behind a config flag, default off.
  > VERIFY: `/ask` latency baseline unchanged; `/discover` returns explainable
  > multi-hop results with evidence.

- [ ] **W5 Explanation payload.** Each `/discover` result answers Why / How /
  Evidence / Confidence / Graph-path — the P4 "explainable" contract, but only
  for this one strategy.
  > VERIFY: response contains all 5 explanation fields, each traceable to real
  > explicit edges.

### Explicitly OUT of the 1-week slice
LLM hypothesis generation, embedding/temporal/event-flow candidate sources,
intent detection + auto strategy selection, evidence graph, contradiction/
stale/missing detection, continuous re-evaluation, graph versioning, tree-sitter
& new ingestion connectors, cross-repo entity resolution.

### Why this slice is the right 1-week bet
- Uses graphs we ALREADY have (fraud, enterprise) — no new ingestion needed.
- 100% deterministic → satisfies "deterministic before LLM" + "explainable" +
  "traceable to evidence" + "never sacrifice integrity for recall".
- Isolated behind a new endpoint + flag → zero risk to the shipped v1.0 `/ask`.
- Produces a genuine, demoable answer to the North-Star question
  "what hidden dependencies exist between these services?" for real.

3. v2.0 (multi-month): full P3–P9 program, with constraint #1 formally lifted,
   scoped phase-by-phase with its own roadmap + acceptance gates.

## Answer to "feasible in the next 1 week?"
- Full 9-phase vision: **No** — multi-month, and conflicts with v1.0's
  zero-drift constraint (so it's v2.0, not a v1.0 add-on).
- A meaningful, honest proof-of-thesis slice (W1–W5): **Yes** — deterministic
  implicit-dependency discovery with explanations, on existing fraud/enterprise
  graphs, behind an opt-in endpoint, in ~1 week.
