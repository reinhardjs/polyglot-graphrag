## Notes

- R2 sits exactly at the 80% target (5 misses out of 25). All misses were
  content-derived queries that retrieved a *related* sibling doc (e.g.
  `fee-schemes.md` query → `README.md`/`backoffice-field-guide.md`) rather
  than the exact source doc. This is acceptable for a broad KB (the related
  doc is still topically correct); if stricter exact-match is wanted later,
  raise TARGET_ACCURACY or tune the keyword/vector entry blend.
- The `--n 2` synth p95 (2.9s) edges the 3s target only because the
  benchmark hammers the shared E2B server with 20 back-to-back calls
  (extraction+synthesis contention). Real single-user latency is 0.7–1.2s.
